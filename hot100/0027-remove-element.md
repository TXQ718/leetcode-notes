# LeetCode 27. 移除元素

## 题目信息

- 题号：27
- 题目：移除元素
- 难度：简单
- 类型：数组、双指针
- 链接：https://leetcode.cn/problems/remove-element/

---

## 题目描述

给定一个数组 `nums` 和一个整数 `val`，要求原地移除数组中所有等于 `val` 的元素，并返回移除后数组的新长度。

题目只要求保证数组前 `k` 个元素是保留下来的元素，后面的元素不重要。

---

## 示例

```text
输入：nums = [3, 2, 2, 3], val = 3
输出：2
解释：移除所有 3 后，数组前两个元素应该是 [2, 2]，返回新长度 2。
```

---

## 解题思路

这道题的核心思路是：

1. 使用变量 `k` 表示新数组的长度，也表示下一个可以写入的位置。
2. 遍历数组中的每个元素，如果当前元素不等于 `val`，就把它放到 `nums[k]`。
3. 每保留一个元素，就让 `k += 1`，最后返回 `k`。

可以从下面几个角度思考：

- 这道题考察数组原地修改。
- 可以使用双指针思想。
- `num` 用来遍历原数组，`k` 用来记录新数组位置。
- 需要注意不能一边遍历数组，一边随意删除数组元素，否则容易出错或效率较低。

---

## 初始想法

我一开始想到的方法是：

```text
只要 val 还在 nums 中，就使用 nums.remove(val) 删除它。
```

代码如下：

```python
while val in nums:
    nums.remove(val)
return len(nums)
```

这种方法可以通过，但效率不高。

原因是：

```python
val in nums
```

需要查找一次。

```python
nums.remove(val)
```

也需要查找并删除一次。

如果 `val` 出现很多次，整体时间复杂度可能接近 `O(n²)`。

因此更推荐使用双指针写法。

---

## Python 代码

```python
class Solution:
    def removeElement(self, nums: List[int], val: int) -> int:
        k = 0

        for num in nums:
            if num != val:
                nums[k] = num
                k += 1

        return k
```

---

## 代码解释

### 1. 关键代码一

```python
k = 0
```

解释这段代码的作用。

`k` 有两个含义：

```text
1. 表示新数组的长度
2. 表示下一个可以写入的位置
```

一开始还没有保留任何元素，所以 `k = 0`。

---

### 2. 关键代码二

```python
for num in nums:
```

解释这段代码的作用。

这行代码表示遍历数组中的每一个元素。

这里使用 `num` 表示当前元素。

不建议写成：

```python
for val in nums:
```

因为函数参数中已经有一个 `val`，表示要删除的目标值。

如果循环变量也叫 `val`，会覆盖原来的变量含义。

---

### 3. 关键代码三

```python
if num != val:
    nums[k] = num
    k += 1
```

解释这段代码的作用。

如果当前元素 `num` 不等于要删除的值 `val`，说明这个元素应该保留。

于是把它放到 `nums[k]` 的位置。

然后执行：

```python
k += 1
```

表示已经多保留了一个元素，下一个保留元素应该放到下一个位置。

---

## Python 语法总结

这道题中用到的 Python 语法有：

### 1. 获取列表长度

```python
len(nums)
```

解释：

`len(nums)` 表示获取列表长度。

类似 C++ 中的：

```cpp
nums.size()
```

---

### 2. 判断元素是否在列表中

```python
val in nums
```

解释：

判断 `val` 是否存在于列表 `nums` 中。

例如：

```python
3 in [3, 2, 2, 3]
```

结果是：

```text
True
```

---

### 3. 删除列表元素

```python
nums.remove(val)
```

解释：

删除列表中第一个等于 `val` 的元素。

注意：`remove()` 每次只删除第一个匹配元素，不会一次删除全部。

---

## 易错点

1. `remove()` 每次只删除第一个匹配元素。
2. `while val in nums` 写法虽然简单，但是效率较低。
3. 不建议写 `for val in nums`，因为会覆盖参数 `val`。

---

## 边界情况

需要注意的特殊情况：

1. 输入为空：如果 `nums = []`，循环不会执行，返回 `0`。
2. 只有一个元素：如果等于 `val`，返回 `0`；否则返回 `1`。
3. 有重复元素：可以正确保留所有不等于 `val` 的元素。
4. 结果不存在：如果所有元素都等于 `val`，返回 `0`。
5. 其他特殊情况：如果没有元素等于 `val`，返回原数组长度。

---

## 复杂度分析

时间复杂度：`O(n)`

原因：

双指针写法只遍历数组一次。

空间复杂度：`O(1)`

原因：

在原数组上修改，只使用了常数个额外变量。
